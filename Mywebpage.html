Lesson 1: Getting Started With HTML

The World Wide Web is made up of many computers communicating with each other. When you open a web page your computer sends a request to the server. The server finds the HTML document and sends it back to the computer. Your web browser then inturprets it and opens it. For a great overview watch this video.

HTML

HTML stands for Hyper Text Markup Language, it is the language computers "speak" to each other in. HTML Documents are made up of text (what you see) and markup (how it looks). HTML documents form the majority of content on the web.

HTML Vocabulary

Tags An HTML tag is always contained within angled brackets. Most have opening and closing brackets, but some are void tags, meaning they do not require a closing bracket.

Elements This refers to everything within the Tag.

Attribute the property of the HTML element

Inline V Block Elements

HTML elements are either Inline or Block Elements. A block element forms a box around what's inside it.

Example of HTML Tags

<name>contents</name> e.g <b>This tag makes a word bold</b>

<tagname ATTR="Value">content</tagname> e.g <a href="www.Twitter.com">This Tag links to Twitter</a>

Computers Are Stupid

Computers can do many wonderful things, but they inturpret all instructions literally. This means programmers need to write exactly what they mean or huge problems can occur.

You Can't Remember Everything

If you forget how to do something, like add a link to a web page, you can always look it up!

Lesson 2: Creating a Web Page with HTML

HTML, CSS and Java Script

These are the languages that programmers use to develop web pages. If we think of the web like a house then HTML is the structure (the rooms) CSS is the design (colours, paintings) and Java script is like the interactive elements (the TV remote). Each language has it's own set of rules. Progammers must follow these rules so that the computer can understand how the web page should be displayed.

The DOM Tree

The Dom (Document Object Model) Tree refers to the fact that an HTML document resembles a tree. An HTML element can have another HTML element nested inside it and so on and so on, all branching out from the same starting point, like the branches of a tree.

When you look at the HTML of a web page, the more deeply indented an element is, the more deeply nested it is.

Boxes

Everything on a web page is in a box! Although you might not be able to see the box. You can use place holder content to remmember which box is which.

Text Editors

There are different text editors programmers can use for writing code (Scratchpad, Codepen, Sublime) they all have different advanatages. Text editors can make programmers lives easier by automatically adding closing tags.

Lesson 3: Adding CSS Style and HTML Structure

Adding Style

You need to tell the browser where to find the styling information.

  • Begin with <!Doctype html> this tells the browser that the type of documnet is an HTML file.
  • Next add <HTML> because everything is written in HTML. Don't forget to include the end tag!
  • Next add the <head></head> tag. This is where you will link to the style file.
  • Next add the <body></body> which is where your text will go.
  • For usability indent your code This video demonstrates the above.

Don't repeat yourself

CSS (Cascading Style Sheets) allows programmers to avoid repition. You can have several style sheets which cascade down until you find the best one to describe the element. Inheritance refers to the fact that properites are applied not only to a specified element but to it's descendants.

Using CSS

First choose a selector, this is the element the CSS will apply to. If you choose an HTML tag the rule will apply to all tags on that page. The other way is to select elements by class. The beginning of this video shows you how to write that code. This web page also gives a great overview of different types of selectors and how to use them.

The Box Model

As we said earlier everything on the web is inside a box. Each box also has padding, a boarder and a margin surrounding it.

These extra elements mean that it's diffcult to calucate how much space the box will take up. To fix this you can use the following CCS styling
* {
     Box-sizing: boarder-box;
}

However not all browsers recongize this code so you need to add
*{
       -webkit-box-sizing:
      border-box;
       -moz-box-sizing:
       border-box;
       box-sizing:
       border-box;        
}

Or you can set the size in % rather than pixals. When using a % the box will change depending of the size of the screen being used.

The Flex Box

CSS can give you power over the positioning of the boxes. One way to do this is to use the Flex Box. When you change the display attribute of the parent container to value flex, the child elements try to align. Here is a guide to Flex Boxes.

Adding Images

When adding an image remember to add alt="This is a screenshot" incase the photo does not display.

  2. Look for repeated styles and semantic elements.
  3. Write your HTML
